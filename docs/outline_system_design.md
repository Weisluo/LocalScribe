# 大纲系统 UI 设计文档

## 1. 概述

### 1.1 设计目标
为 LocalScribe 编辑器新增大纲界面，与现有世界观设定界面采用相同的内嵌式设计模式，提供三个子页面：故事发展链路图、卷大纲编辑、章节大纲编辑。

### 1.2 设计原则
- **一致性**：与现有世界观设定界面保持视觉和交互一致性
- **层级清晰**：卷-幕-章三级结构清晰展示，支持折叠展开
- **实时同步**：与目录树数据保持同步，自动生成层级结构
- **沉浸式编辑**：提供类似主页编辑器的纸张文本框体验

### 1.3 入口位置
在左侧栏「世界观设定」按钮下方新增「大纲」入口按钮

## 4. 子页面详细设计

### 4.1 故事发展链路图 (StoryChainView)

#### 4.1.1 功能描述
展示故事发展的时间线视图，以卷-幕为层级，每一幕包含一个从上往下的事件流程图。

#### 4.1.2 数据结构
```typescript
// 连接类型枚举
enum ConnectionType {
  DIRECT = 'direct',       // 直接连接（线性发展）
  BRANCH = 'branch',       // 条件分支（二选一/多选一）
  PARALLEL = 'parallel',   // 并行开始
  MERGE = 'merge',         // 汇合点
  LOOP = 'loop',           // 循环/回溯
  JUMP = 'jump',           // 跳跃（闪回/闪前）
}

// 连接关系
interface EventConnection {
  id: string;
  from: string;            // 源事件ID
  to: string;              // 目标事件ID
  type: ConnectionType;
  label?: string;          // 连接标签（如"选择A"、"条件：成功"）
  condition?: string;      // 条件描述（分支时显示）
  style?: {
    color?: string;        // 线条颜色
    dashed?: boolean;      // 是否虚线
    thickness?: number;    // 线条粗细
  };
}

// 事件卡片数据
interface StoryEvent {
  id: string;
  actId: string;           // 所属幕ID
  title: string;           // 事件标题
  content: string;         // 事件内容简述
  characters: string[];    // 参与角色ID列表
  location?: string;       // 地点
  timestamp?: string;      // 时间标记
  order: number;           // 排序（用于初始布局）
  
  // 连接关系（支持多对多）
  outgoingConnections: EventConnection[];  // 出向连接
  incomingConnections: EventConnection[];  // 入向连接（计算属性）
  
  // 视觉属性
  position?: {
    x: number;             // 相对坐标（自动计算）
    y: number;
    lane?: number;         // 泳道编号（用于并行线）
  };
  
  // 事件类型
  eventType?: 'normal' | 'decision' | 'milestone' | 'flashback' | 'flashforward';
}

// 幕链路数据
interface ActChain {
  actId: string;
  actName: string;
  volumeId: string;
  events: StoryEvent[];
  connections: EventConnection[];  // 幕内所有连接
  lanes: number;                   // 泳道数（用于并行布局）
}

// 卷链路数据
interface VolumeChain {
  volumeId: string;
  volumeName: string;
  order: number;
  acts: ActChain[];
}
```

**时间轴数据结构**

```typescript
interface TimelineNode {
  eventId: string;
  timestamp: string;          // 显示的时间标签 (如 "Day 1", "第三章", "2024-01-15")
  absoluteTime?: string;      // 绝对时间 (可选)
  relativeTime?: number;      // 相对时间 (用于计算间距)
  nodeType: 'normal' | 'decision' | 'milestone' | 'flashback' | 'flashforward';
  position: {                 // 在画布中的位置
    x: number;
    y: number;
  };
}

interface TimelineConfig {
  visible: boolean;           // 是否显示时间轴
  width: number;              // 时间轴宽度 (默认 80px)
  position: 'left' | 'right'; // 位置
  showRelativeTime: boolean;  // 显示相对时间差
  autoCalculate: boolean;     // 自动计算时间间距
}
```

**时间轴交互**

| 操作 | 行为 |
|------|------|
| 点击时间节点 | 选中对应事件，高亮显示 |
| 悬停时间节点 | 显示时间详情 tooltip |
| 拖拽时间节点 | 调整时间顺序（重新排序事件） |
| 双击时间节点 | 编辑时间标签 |
| 滚轮 | 时间轴与画布同步滚动 |

**时间轴与事件联动**

- 事件移动时，时间节点跟随移动
- 时间标签支持自定义（如 "Day 1", "第三章", "清晨"）
- 闪回/闪前节点显示相对时间（如 "[7天前]"）
- 时间跨度大的事件自动压缩间距

**事件卡片 (EventNode)**

基础样式：
- 尺寸: 宽度 200px，高度自适应（最小 80px，最大 160px）
- 背景: `bg-card` 带微妙阴影 `shadow-sm`
- 边框: `border border-border/60`，hover 时 `border-accent/50`
- 圆角: `rounded-lg`
- 内边距: `p-3`

根据事件类型的不同样式：

| 事件类型 | 边框颜色 | 图标 | 特殊标记 |
|---------|---------|------|---------|
| normal (普通) | border-border/60 | Circle | 无 |
| decision (决策) | border-amber-400 | GitBranch | 右上角显示分支图标 |
| milestone (里程碑) | border-purple-400 | Flag | 左侧彩色条 |
| flashback (闪回) | border-blue-400 | History | 虚线边框 |
| flashforward (闪前) | border-green-400 | FastForward | 虚线边框 |

内容结构:
- 标题: 14px font-semibold，单行截断
- 内容摘要: 12px text-muted-foreground，最多2行
- 元信息行: 11px 图标+文字，flex 布局，溢出隐藏
  - 角色图标 + 角色名称列表
  - 地点图标 + 地点名称

**连接线 (ConnectionLine)**

SVG 绘制，根据连接类型显示不同样式：

| 连接类型 | 线条样式 | 箭头 | 标签位置 |
|---------|---------|------|---------|
| DIRECT | 实线 2px | 末端箭头 | 无 |
| BRANCH | 实线 2px | 末端箭头 | 线上方居中 |
| PARALLEL | 双线 各1.5px | 双箭头 | "并行开始"/"并行结束" |
| MERGE | 实线 2px | 倒三角箭头 | 汇合点上方 |
| LOOP | 虚线 2px | 弧形箭头 | "回溯"/"循环" |
| JUMP | 点线 2px | 闪电箭头 | 时间标记 |

线条颜色:
- 默认: `stroke-muted-foreground`
- 选中: `stroke-accent`
- 条件分支: 根据条件显示不同颜色（成功=绿色，失败=红色等）

**泳道 (Lane)**
- 背景: 交替的微妙色块 `bg-muted/5`
- 分隔线: 虚线 `border-l border-dashed border-border/30`
- 最小宽度: 240px
- 自动计算所需泳道数

**底部工具栏**
```
┌─────────────────────────────────────────────────────────────┐
│  [+ 添加事件]  [连接模式: 关闭 ▼]  [自动布局 ▶]  [适应画布]    │
└─────────────────────────────────────────────────────────────┘
```

- 添加事件: 在当前选中事件后添加，无选中则在末尾添加

**自动布局算法**

从上往下自动排列事件：

1. **层级计算**: 根据连接关系计算每个事件的层级（深度）
2. **泳道分配**: 
   - 并行事件分配到不同泳道
   - 分支事件在汇合前保持独立泳道
   - 尽量保持最少泳道数
3. **位置计算**:
   - Y坐标 = 层级 × 层间距（120px）
   - X坐标 = 泳道中心
4. **连线优化**: 使用贝塞尔曲线，避免交叉

**自动吸附 (Snap)**

吸附规则：
- **事件卡片**: 吸附到最近的网格点（以卡片中心为准）
- **吸附距离**: 20px（超过此距离不吸附）
- **吸附动画**: 200ms ease-out

吸附优先级：
1. 网格点吸附（基础）
2. 对齐线吸附（增强）
   - 与其他事件左对齐
   - 与其他事件居中对齐
   - 与其他事件右对齐
   - 与其他事件顶部/底部对齐

**渐进式加载**

分阶段显示内容（根据数据量动态调整）：

**少量数据（< 20 个事件）- 快速模式：**

| 阶段 | 时间 | 显示内容 |
|------|------|---------|
| 1 | 0-50ms | 骨架屏显示（极短） |
| 2 | 50-150ms | 卷/幕结构 + 时间轴节点 |
| 3 | 150-300ms | 事件卡片 + 连接线绘制完成 |
| 4 | 300ms | 骨架屏淡出，内容完全显示 |

**中等数据（20-50 个事件）- 标准模式：**

| 阶段 | 时间 | 显示内容 |
|------|------|---------|
| 1 | 0-100ms | 骨架屏显示 |
| 2 | 100-300ms | 卷/幕结构 + 时间轴节点 |
| 3 | 300-600ms | 事件卡片逐个显示 |
| 4 | 600ms | 连接线绘制完成 |

**大量数据（> 50 个事件）- 渐进模式：**

| 阶段 | 时间 | 显示内容 |
|------|------|---------|
| 1 | 0-200ms | 骨架屏显示 |
| 2 | 200-500ms | 卷/幕结构 + 当前视口时间轴 |
| 3 | 500-800ms | 当前视口事件卡片 |
| 4 | 800ms+ | 视口外内容懒加载 |

**加载策略**

```typescript
interface LoadingStrategy {
  // 根据事件数量选择加载策略
  getStrategy(eventCount: number): {
    showSkeleton: boolean;      // 是否显示骨架屏
    skeletonDuration: number;   // 骨架屏显示时长
    staggerDelay: number;       // 逐个显示延迟
    enableVirtualization: boolean; // 是否启用虚拟滚动
  };
}

// 策略配置
const loadingStrategies = {
  fast: {        // < 20 事件
    showSkeleton: true,
    skeletonDuration: 50,      // 50ms 骨架屏
    staggerDelay: 10,          // 10ms 间隔
    enableVirtualization: false,
  },
  standard: {    // 20-50 事件
    showSkeleton: true,
    skeletonDuration: 100,     // 100ms 骨架屏
    staggerDelay: 20,          // 20ms 间隔
    enableVirtualization: false,
  },
  progressive: { // > 50 事件
    showSkeleton: true,
    skeletonDuration: 200,     // 200ms 骨架屏
    staggerDelay: 30,          // 30ms 间隔
    enableVirtualization: true,
  },
  instant: {     // 极快模式（本地缓存）
    showSkeleton: false,       // 不显示骨架屏
    skeletonDuration: 0,
    staggerDelay: 0,
    enableVirtualization: false,
  },
};
```

**优化策略**

1. **预加载**：鼠标悬停在「大纲」按钮时开始预加载数据
2. **本地缓存**：已加载的数据缓存，再次打开时直接显示（无骨架屏）
3. **骨架屏阈值**：加载时间 < 100ms 时不显示骨架屏，避免闪烁
4. **渐进淡出**：骨架屏淡出动画 150ms，与新内容淡入同时进行

  config, 
  onLoadComplete 
}: { 
  config: SkeletonConfig;
  onLoadComplete?: () => void;

  const [progress, setProgress] = useState(0);
  
  useEffect(() => {
    // 模拟渐进加载

      { time: 200, progress: 20 },
      { time: 500, progress: 40 },
      { time: 1000, progress: 70 },
      { time: 1500, progress: 100 },
    ];
    
    stages.forEach(({ time, progress }) => {
      setTimeout(() => setProgress(progress), time);
    });
    
    setTimeout(() => onLoadComplete?.(), 1500);
  }, []);
  
  return (
    <div className="skeleton-container">
      {config.showTimeline && <TimelineSkeleton />}
      {config.showEvents && <EventSkeleton count={config.eventCount} />}
      {config.showConnections && <ConnectionSkeleton />}
      {config.showToolbar && <ToolbarSkeleton />}
      <ProgressBar progress={progress} />
    </div>
  );
}
